试水Pelican

2014年08月04日, 周一 web / pelican

先扯两句

早前也转战过 WordPress（注册了域名），是好用，但是个人业余维护起来还是复杂，不如把时间省出来，多看书码字搬砖陪妹子。
所以最后回归了简单的 Github Pages + Markdown，生成工具出于熟悉 Python 选择了 Pelican 而不是常见的 Jekyll。
就快捷 set up 而言，不到 1 个小时（当然要想调教到合自己的胃口就因人而异了）；就使用上，对着本地而不是网页码字，简直舒畅。不如众乐乐，我也来扯扯。

多多搜索

鉴于目前只是小马过河，就专注于一些使用时遇到的细节，不弄整理一步步的流程了。新手如我，可以提供的参考资料主要有 （当然之后随着版本更新，会有更多适合你的）：

  1. 必须致敬的 理想的写作环境：Git+Github+Markdown+Jekyll。最早这么做的人一定不是他，但扫了一些经验博文，大多从 WordPress 转过来的人都看过这篇。可见 WordPress 从某种程度上的坑，以及心理科班的说服力。
  2. 一步一步打造 Geek 风格的技术博客 ，作者分享精神极赞，把整个网站 make 前的资料尤其是 pelicanconf.py 也放出来。翻翻看能学会很多。
  3. 搭建一个免费的，无限流量的 Blog----github Pages 和 Jekyll 入门 ，我用的不是 Jekyll, 但是这篇太亲和，以致差一点就倒向了 Jekyll。
  4. Play with Jekyll, 有许多可借鉴的地方。
  5. Pelican 官方文档 , 且要翻一阵子这个。。。
  6. 用 Pelican 和 GitHub Pages 搭建免费的个人博客 ，中文 Pelican 的一个不错的总结。但是我想说。。。博主你这么快就不更新了是要做反例么！！补一句，那个 theme 并不是 bootstrap3, 而是 tuxlite_tbs。配色赞的很，上面的 一步一步打造 Geek 风格的技术博客 用的也是这个。如果驾驭不了 bootstrap3 的话，我说不定也投奔它了 XD。
  7. 如果熟悉英文，还可以参考 从 WP 到 Jekyll 到 Pelican 的感想 ， 实践 ，以及 Disqus。

下水后

  • 日期格式？
    如下。其中 %Y 年 %m 月 %d 日 , 周 %a 可以自由搭配。

    TIMEZONE = 'Asia/Shanghai'
DEFAULT_LANG = u'zh'
DATE_FORMATS = {
    'zh':('zh_CN','%Y 年 %m 月 %d 日 ,  周 %a'),
}

  • Disqus 无法成功加载？
    把配置文件中的 SITEUR = “” 改成自己的地址。关闭 RELATIVE_URLS。

  • 抬头 Menu 要 About，不要 Category？
    About 的源文件要放在 ./content/pages 文件夹里。DISPLAY_CATEGORIES_ON_MENU = False 可以关闭 Category 的显示

  • 找不到关于 Google 站内搜索的教程？
    原博客挂了。可以参考这里：Pelican 添加自定义搜索 。
    或者看 lizherui 的文件 （添加 search.html 放置搜索输出 , 修改相应的 template、css 和配置文件）。
    我的 theme 排版不同，暂时不添加这个。

  • FILES_TO_COPY 失效？
    已经在新版本中弃用，采用了 EXTRA_PATH_METADATA

  • make html 还是 make publish？
    翻了翻 Makefile。我这里它们是一样的，取决于你用的是 pelicanconf.py 还是 publishconf.py。甚至你可以自己定义一个文件作配置。岔开来说，能感觉到玩腻 配置 后，下一步就该玩 Makefile 了。

  • 为啥加载速度那么慢？
    可以看看 chrome 的 F12 中 network 相关的加载速度（Ctrl+F5 强制刷新刷）。一般来说，优先考虑主题中某些比如谷哥的服务被挡，或者是字体文件太大（本身 github 在国内的速度也不乐观）。前者更为制约。所以要么通过下载相应的文件并在主题里进行修改，要么通过国内的服务商提供的常用前端公共库，要么放弃该字体。具体可以参考 这篇日志 。
