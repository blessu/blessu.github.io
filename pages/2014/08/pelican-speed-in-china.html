建站加载慢，可以这么办

2014年08月18日, 周一 web / pelican

其实不局限于 Pelican。至少我看到的大部分相似问题来自 WordPress。

Pelican 建好站后，就期待着这一亩三分地慢慢成长。But, 每次自己刷页面看效果时都要 20 多秒才搞定。自家的娃还能忍，别人来看估计就没这个耐心。所以抽空把这个不爽解决了。

原因出在为了字体文件，每次进入网站都需要尝试下载托管的字体。有些主题使用的是 google 服务的 Open Sans 字体，而谷哥近来被搞得比较惨。所以在 chrome 下看 F12 就会有这样的情景：

这 20 秒就只能无奈地面对着图片和页面框架。
解决办法自然是从字体文件的获取入手。具体操作因人而异，主要有三个思路：

  1. 下载字体文件，然后托管到网站的静态文件夹
    • 方法：可以参考 让 wordpress 主题绕开对 google 的依赖 。
    • 优点：自己动手，丰衣足食。国内外连接速度都还好。
    • 缺点：从 Github 到本地的下载时间也略有延迟。不过感谢码农们的争取，相比谷哥很可以接受了。
  2. 利用国内的公共库，把主题中相应对字体的引用调整到速度快的某些网站。
    • 方法：可以参考 某前端公共库 CDN 服务 ，当然其他很多大户都有类似的服务。
    • 优点：只要在国内，速度很爽。改起来很省事，值得表扬。
    • 缺点：吃人嘴短， 没节操一阵子 ，扩展性弱一点。国外连接速度没试，未知。
  3. 老子不用 Open Sans 了。
    • 方法 : 直接封印了 @import 字体那部分。甚至可以考虑换个主题 XD
    • 优点：Open Sans 和中文没啥关系，影响不大。网站放哪都不影响速度。
    • 缺点：想了想，好像除了 “ 一般来说主题自己对应的字体比较搭调 ” 外没啥可反驳的。

中文博客短期速成推荐方法二，长期解决办法大致是：
1. 整个网站托管到国内的某些 github 类似站上，然后采取上面的方法一。需要暂时观望一段时间。
2. 方法三。

举个例子：bootstrap3 主题的解决方法

bootstrap3 主题的比较特殊的字体包括谷哥的 Open Sans 和美化 “ 图标 ” 的 font-awesome。

之前 暂时 采取过 方法二 ，两步完成。
1. 根据 某前端公共库 CDN 服务 ，把 css 第一行的 googleapis 改成 useso；
2. 把主题模板 pelican-bootstrap3/templates/base.html 中 href="{{ SITEURL }}/theme/css/font-awesome.min.css" 改成 href="http://libs.useso.com/js/font-awesome/4.1.0/css/font-awesome.css"

今天比较了一下效果，觉得 Open Sans-> 默认的英文字体还可以接受。于是采取 方法三 ，顺便把中文字体从默认换成了微软雅黑和 STHeiti。
1. 首先封印掉 css 第一行的 @import 字体那部分。
2. 把含有 "Open Sans",...,sans-serif 的部分换成 STHeiti,...,"Microsoft YaHei", 微软雅黑 ,sans-serif ，同时调大 body 字号到 16px。
3. 注意 "Microsoft YaHei", 微软雅黑 要在所有英文字体之后，默认字体之前。否则，似乎雅黑会自动把单引号转成全角，难看到爆。

  • 扩展阅读 1: 关于浏览器默认字体

  • 扩展阅读 3: css 设置各种中文字体如雅黑、黑体、宋体、楷体等等
  • 引自 让 wordpress 主题绕开对 google 的依赖

    网站中添加的谷歌 analytics 的 js 代码不会对网页加载产生影响。因为它是动态写上去，等网站发起对 ga.js 的请求的时候，网页中该请求的东西都请求完了。所以不会影响，只是有墙的情况下，由于 ga.js 请求不到，会影响 google analytics 对你网站流量监控的准确性。
