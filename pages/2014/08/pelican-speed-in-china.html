建站加载慢，可以这么办

2014年08月18日, 周一 web / pelican

其实并不局限于 Pelican。至少我看到的大部分相似问题来自 WordPress。

Pelican 建好站后，就期待着这一亩三分地慢慢成长。But, 每次自己刷页面看效果时都要 20 多秒才搞定。自家的娃还能忍，别人来看估计就没这个耐心。所以抽空把这个不爽 暂时 解决了。

原因出在为了字体文件，每次进入网站都需要尝试下载托管的字体。有些主题使用的是 google 服务的 Open Sans 字体，而谷哥近来被搞得比较惨。所以在 chrome 下看 F12 就会有这样的情景：

这 20 秒就只能无奈地面对着图片和页面框架。
解决办法只能是从字体文件的获取入手。解决方法因人而异，我这里主要有三个思路：

  1. 把字体文件下下来，然后托管到网站的静态文件夹
    • 方法：可以参考 让 wordpress 主题绕开对 google 的依赖 。
    • 优点：自己动手，丰衣足食。国内外连接速度都还好。
    • 缺点：从 Github 到本地的下载时间也会有延迟，虽然感谢码农们的争取，很可以接受了。
  2. 利用国内的公共库，把主题中相应对字体的引用调整到速度快的某些网站。
    • 方法：可以参考 某前端公共库 CDN 服务 ，当然其他很多大户都有类似的服务。
    • 优点：只要在国内，速度很爽。改起来很省事，值得表扬。
    • 缺点：吃人嘴短， 暂时 没节操一阵子。国外连接速度没试，估计不咋样（当然定位本来就是中文博客，无所了）。
  3. 破罐子破摔，老子不用 Open Sans 了。
    • 方法 : 直接封印了导入字体那部分。甚至可以考虑换个主题 XD
    • 优点：Open Sans 和中文没太大关系，影响不大。放哪都不太影响速度。
    • 缺点：想了想，好像除了 “ 一般来说主题自己对应的字体还是比较搭调的 ” 外没啥可反驳的。

呃，我 暂时 采取的是第二种。中文博客的长期解决办法估计有二：
1. 整个网站托管到国内的某些 github 类似站上，然后采取上面的方法 1。需要暂时观望一段时间。
2. 换个字体 / 换个主题。

另外

  • bootstrap 主题的字体位置在 ./pelican-bootstrap3/static/css/bootstrap.spacelab.min.css 的第一行。
  • 引自 让 wordpress 主题绕开对 google 的依赖

    网站中添加的谷歌 analytics 的 js 代码不会对网页加载产生影响。因为它是动态写上去，等网站发起对 ga.js 的请求的时候，网页中该请求的东西都请求完了。所以不会影响，只是有墙的情况下，由于 ga.js 请求不到，会影响 google analytics 对你网站流量监控的准确性。
