试水Pelican

Date 2014年08月04日, 周一 Tags web

先前也转战过 WordPress（注册了域名），是好用，但是个人业余维护起来还是略复杂，不如把时间省出来多看书搬砖陪妹子。

所以最后回归了简单的 Github Pages + Markdown，生成工具出于熟悉 Python 选择了 Pelican 而不是常见的 Jekyll。

就 set up 而言，看着 Pelican + Github 搭建自己的静态博客 来大概不到 1 个小时；就使用上，对着本地而不是网页码字，简直舒畅。

熟悉了应该会修改这篇，现在还是笔记和测试，便不那么规整。目前可以提供的参考资料有：

  1. 这类博文必须致敬的 理想的写作环境：Git+Github+Markdown+Jekyll。最早这么做的人一定不是他，但扫了一些经验博文，大多从 WordPress 转过来的人都看过这篇。可见 WordPress 从某种程度上的坑，以及心理科班的说服力。
  2. 搭建一个免费的，无限流量的 Blog----github Pages 和 Jekyll 入门 ，我用的不是 Jekyll, 但是这篇太亲和，以致差一点就倒向了 Jekyll。
  3. Play with Jekyll, 有许多可借鉴的地方。
  4. Pelican Quickstart, 且要翻一阵子这个文档。。。
  5. 用 Pelican 和 GitHub Pages 搭建免费的个人博客 ，中文 Pelican 的一个不错的总结。但是我想说。。。博主你这么快就不更新了是要做反例么！！补一句，那个 theme 并不是 bootstrap3, 而是 tuxlite_tbs。配色赞的很，如果驾驭不了 bootstrap3 的话说不定我也投奔它了 XD
  6. 如果熟悉英文，还可以参考 从 WP 到 Jekyll 到 Pelican 的感想 ， 实践 ，以及 Disqus。
